Maze generation is just an excuse

...or how to try 10 different tools you never used before

NOTE: You may see this submission twice, as I submitted before but it was linking to first blog-post only. Now it is a little bit more.

Initially I just wanted an excuse to use Fable, Webpack, Node, Jade, Virtual DOM, React, and... everything. In general, to use all the tools I would like to try and have small chance to use in my day-job.

So I implemented maze generator using DFS in Fable. It was generating SVG using VirtualDOM. Everything was packaged with Webpack. The performance wasn't good, so I switched to jCanvas.

Never used Fable before... check. Never used VirtualDOM before... check. Never used Webpack before... check. Never used jCanvas before... check. Cool.

It was supposed to be it, but I actually found it quite interesting to explore this X-to-JavaScript transpilers. So I also tried Scala.js and Kotlin. I also decided to use different algorithm per language.

I do understand that some of the competitors might have implemented multidimensional, infinite, fractal mazes, so my solutions can be quite humble, but what I wanted to achive was to evaluate those transpilers and I did. From this perspective mission has been acomplished. I also read a tweet recently that (quoting from memory) "coding challenges help find people with no family or day-job". I have both, so my resources were quite limited.
